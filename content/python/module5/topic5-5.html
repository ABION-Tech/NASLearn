Page 5: Best Practices for Writing Functions

1. Use Descriptive Function Names

Function names should clearly indicate their purpose to improve readability.

        
# Bad Example
def fx(x):
return x * 2

# Good Example
def double_number(x):
return x * 2

2. Keep Functions Short and Focused

Each function should perform a single, well-defined task.

        
# Bad Example: Too many responsibilities
def process_data(data):
cleaned = clean_data(data)
transformed = transform_data(cleaned)
saved = save_to_database(transformed)
return saved

# Good Example: Separate concerns
def clean_data(data):

return cleaned_data

3. Use Default Parameter Values Wisely

Provide default values for parameters when appropriate.

        
def greet(name="User"):
print(f"Hello, {name}!")

greet()  # Output: Hello, User!
greet("Ayo")  # Output: Hello, Ayo!

4. Use Type Hints for Clarity

Type hints help other developers understand expected input and output types.

        
def add_numbers(a: int, b: int) -> int:
return a + b

5. Avoid Using Global Variables

Global variables can cause unexpected side effects and should be avoided in most cases.

        
# Bad Example
counter = 0

def increment():
global counter
counter += 1
return counter

# Good Example
def increment(counter):
return counter + 1

6. Write Docstrings for Documentation

Docstrings help explain what a function does, its parameters, and its return value.

        
def calculate_area(radius: float) -> float:
"""
Calculate the area of a circle.

Parameters:
    radius (float): The radius of the circle.

Returns:
    float: The calculated area.
"""
return 3.1416 * radius ** 2

7. Handle Exceptions Gracefully

Use try-except blocks to handle errors properly.

        
def divide(a: int, b: int) -> float:
try:
    return a / b
except ZeroDivisionError:
    return "Error: Cannot divide by zero"
except TypeError:
    return "Error: Invalid input type"

print(divide(10, 2))  # Output: 5.0
print(divide(10, 0))  # Output: Error: Cannot divide by zero

8. Follow the DRY Principle (Don't Repeat Yourself)

Instead of repeating code, refactor common logic into reusable functions.

        
# Bad Example: Repetitive Code
def area_of_rectangle(width, height):
return width * height

def area_of_triangle(base, height):
return 0.5 * base * height

# Good Example: Generalized Function
def calculate_area(shape: str, dimension1: float, dimension2: float) -> float:
if shape == "rectangle":
    return dimension1 * dimension2
elif shape == "triangle":
    return 0.5 * dimension1 * dimension2
else:
    return "Invalid shape"

print(calculate_area("rectangle", 10, 5))  # Output: 50
print(calculate_area("triangle", 10, 5))  # Output: 25

9. Test Your Functions

Always test functions to ensure they work as expected.

        
# Example of simple unit tests
def test_add_numbers():
assert add_numbers(2, 3) == 5
assert add_numbers(-1, 1) == 0
assert add_numbers(0, 0) == 0
print("All tests passed!")

test_add_numbers()
