Setting Up a Server

2.1. Introduction to Node.js

Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. It enables developers to use JavaScript for server-side programming.

Installation:

  • Download and install Node.js from nodejs.org.
  • Verify the installation using node -v and npm -v in the terminal.

Basic Node.js Application:

// app.js const http = require('http'); const server = http.createServer((req, res) => { res.statusCode = 200; res.setHeader('Content-Type', 'text/plain'); res.end('Hello, World!\n'); }); server.listen(3000, '127.0.0.1', () => { console.log('Server running at http://127.0.0.1:3000/'); });

Explanation:

  • http.createServer(): Creates a new HTTP server.
  • req: Represents the incoming request.
  • res: Represents the response that will be sent to the client.
  • server.listen(): Binds the server to a specific port and IP address.

2.2. Introduction to Express.js

Express.js is a minimalist web framework for Node.js that simplifies the process of building web applications and APIs.

Installation:

  • Install Express using npm:
    • npm install express

Creating a Basic Express Server:

// app.js const express = require('express'); const app = express(); app.get('/', (req, res) => { res.send('Hello, Express!'); }); app.listen(3000, () => { console.log('Server running on port 3000'); });

Explanation:

  • express(): Initializes an Express application.
  • app.get(): Defines a route for handling GET requests to the root URL.
  • app.listen(): Starts the server and listens on the specified port.

Conclusion

Setting up a server using Node.js and Express.js is a fundamental skill for back-end developers. With Node.js, you can harness the power of JavaScript on the server side, while Express.js provides a robust framework to build efficient web applications. Mastering these tools will enable you to create dynamic, server-side applications that interact seamlessly with front-end technologies.
