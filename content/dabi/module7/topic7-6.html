Project: Retail Sales Analysis and Demand Forecasting

Business Problem

A national retail chain is experiencing inconsistent sales performance across stores and wants to understand the factors driving sales to optimize inventory management and marketing strategies. They need to answer key questions:

  • What factors most significantly impact sales?
  • How effective are their current promotions?
  • Can they accurately forecast future demand to optimize inventory?
  • Which stores are underperforming and why?

Step-by-Step Guide

Objective

Guide

Answer

To extract data from SQL, you would typically use a database connector in Python. Here's a conceptual approach:

import sqlite3
import pandas as pd

# Connect to the database
conn = sqlite3.connect('retail_database.db')

# Write SQL query to extract sales data with relevant joins
query = """
SELECT 
    s.date, 
    s.store_id, 
    s.product_id, 
    s.quantity, 
    s.price,
    p.category,
    p.brand,
    st.location,
    st.size_sqft,
    st.staff_count,
    pr.promotion_type,
    pr.discount_percentage
FROM 
    sales s
JOIN 
    products p ON s.product_id = p.product_id
JOIN 
    stores st ON s.store_id = st.store_id
LEFT JOIN 
    promotions pr ON s.promotion_id = pr.promotion_id
WHERE 
    s.date >= '2020-01-01'
"""

# Execute query and load into DataFrame
df = pd.read_sql(query, conn)

# Close the connection
conn.close()

# Save to CSV for later use
df.to_csv('retail_sales_data.csv', index=False)

Here's how you might approach data cleaning and exploration:

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# Load the data
df = pd.read_csv('retail_sales_data.csv')

# Convert date to datetime
df['date'] = pd.to_datetime(df['date'])

# Check for missing values
print(df.isnull().sum())

# Handle missing values
df['promotion_type'].fillna('No Promotion', inplace=True)
df['discount_percentage'].fillna(0, inplace=True)

# Create total sales feature
df['total_sales'] = df['quantity'] * df['price']

# Extract time features
df['year'] = df['date'].dt.year
df['month'] = df['date'].dt.month
df['day_of_week'] = df['date'].dt.dayofweek
df['is_weekend'] = df['day_of_week'].isin([5, 6]).astype(int)

# Create holiday flag (simplified example)
holidays = ['2020-12-25', '2021-12-25', '2020-07-04', '2021-07-04']  # etc.
df['is_holiday'] = df['date'].dt.strftime('%Y-%m-%d').isin(holidays).astype(int)

# Remove outliers in total_sales (e.g., above 99th percentile)
upper_limit = df['total_sales'].quantile(0.99)
df = df[df['total_sales'] <= upper_limit]

# EDA: Sales over time
plt.figure(figsize=(12, 6))
df.groupby('date')['total_sales'].sum().plot()
plt.title('Total Sales Over Time')
plt.ylabel('Total Sales')
plt.show()

# Sales by category
plt.figure(figsize=(10, 6))
sns.barplot(x='category', y='total_sales', data=df, estimator=sum)
plt.title('Total Sales by Category')
plt.xticks(rotation=45)
plt.show()

# Correlation heatmap
numeric_df = df.select_dtypes(include=[np.number])
plt.figure(figsize=(10, 8))
sns.heatmap(numeric_df.corr(), annot=True, cmap='coolwarm')
plt.title('Correlation Matrix')
plt.show()

Step 3: Test Hypotheses (e.g., "Promotion Y drives higher revenue")

Objective

Statistically validate assumptions about factors affecting sales.

Guide
  1. Formulate Hypotheses:
    • Based on EDA and business knowledge, develop testable hypotheses.
    • Examples:
      • "Promotion Y increases average sales by at least 15%."
      • "Stores in urban areas have higher sales per square foot than suburban stores."
      • "Holiday periods have significantly higher sales than non-holiday periods."
  2. Select Appropriate Tests:
    • For comparing means between groups: t-tests or ANOVA.
    • For relationships between categorical variables: chi-square tests.
    • For correlations: Pearson or Spearman correlation coefficients.
  3. Conduct Statistical Testing:
    • Prepare data for each test (grouping, filtering).
    • Perform the tests and extract relevant statistics (p-values, confidence intervals).
    • Interpret results in the context of the business problem.
  4. Document Findings:
    • Summarize which hypotheses were supported or rejected.
    • Quantify the impact of significant factors (e.g., "Promotion Y increases sales by 18.5%").
    • Identify unexpected findings that warrant further investigation.
Key Considerations
  • Ensure assumptions of statistical tests are met (normality, equal variance, etc.).
  • Consider practical significance alongside statistical significance.
  • Adjust for multiple comparisons if conducting many tests.
Answer

Here's how you might test hypotheses about promotions:

from scipy import stats

# Hypothesis: Promotion Y increases sales
# Group data by promotion type
promo_y = df[df['promotion_type'] == 'Promotion Y']['total_sales']
no_promo = df[df['promotion_type'] == 'No Promotion']['total_sales']

# T-test
t_stat, p_value = stats.ttest_ind(promo_y, no_promo, equal_var=False)
print(f"T-statistic: {t_stat}, P-value: {p_value}")

# Calculate the actual difference
mean_diff = promo_y.mean() - no_promo.mean()
percent_increase = (mean_diff / no_promo.mean()) * 100
print(f"Mean difference: ${mean_diff:.2f}")
print(f"Percentage increase: {percent_increase:.2f}%")

# Interpretation
if p_value < 0.05:
    print("Conclusion: Reject the null hypothesis. Promotion Y significantly increases sales.")
else:
    print("Conclusion: Fail to reject the null hypothesis. No significant evidence that Promotion Y increases sales.")

Step 4: Build a Predictive Model (e.g., Forecast Demand)

Objective

Develop a model to forecast future sales and identify key drivers.

Guide
  1. Define the Modeling Goal:
    • Determine what to predict (e.g., daily sales, weekly sales by store).
    • Establish the prediction horizon (e.g., 30 days, 90 days).
  2. Prepare Data for Modeling:
    • Select relevant features based on EDA and hypothesis testing.
    • Handle time-specific considerations (trends, seasonality).
    • Split data into training and testing sets, preserving temporal order.
  3. Select and Train Models:
    • Start with simpler models (linear regression) as baselines.
    • Progress to more complex models (Random Forest, Gradient Boosting).
    • For time series forecasting, consider ARIMA, Prophet, or LSTM networks.
    • Train multiple models and compare their performance.
  4. Evaluate Model Performance:
    • Use appropriate metrics (MAE, RMSE, MAPE for regression; accuracy, precision, recall for classification).
    • Analyze residuals to identify patterns the model missed.
    • Perform cross-validation to ensure robustness.
  5. Interpret Model Results:
    • Extract feature importance to identify key drivers.
    • Analyze model predictions for insights (e.g., which stores are predicted to underperform).
    • Quantify the business impact of the model (e.g., potential reduction in stockouts).
Key Considerations
  • Balance model complexity with interpretability based on business needs.
  • Consider the cost of different types of errors (false positives vs. false negatives).
  • Document the modeling process for transparency and reproducibility.
Answer

Here's how you might build a sales forecasting model:

from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_absolute_error, mean_squared_error
import numpy as np

# Prepare data for modeling
# Aggregate daily sales
daily_sales = df.groupby('date').agg({
    'total_sales': 'sum',
    'is_weekend': 'max',
    'is_holiday': 'max'
}).reset_index()

# Create lag features
for i in range(1, 8):
    daily_sales[f'lag_{i}'] = daily_sales['total_sales'].shift(i)

# Create moving average features
daily_sales['ma_7'] = daily_sales['total_sales'].rolling(window=7).mean().shift(1)

# Drop rows with NaN values
daily_sales.dropna(inplace=True)

# Define features and target
X = daily_sales.drop(['date', 'total_sales'], axis=1)
y = daily_sales['total_sales']

# Split data (temporal split)
split_point = int(len(daily_sales) * 0.8)
X_train, X_test = X[:split_point], X[split_point:]
y_train, y_test = y[:split_point], y[split_point:]

# Train model
model = RandomForestRegressor(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

# Make predictions
y_pred = model.predict(X_test)

# Evaluate model
mae = mean_absolute_error(y_test, y_pred)
rmse = np.sqrt(mean_squared_error(y_test, y_pred))
print(f"MAE: {mae:.2f}")
print(f"RMSE: {rmse:.2f}")

# Feature importance
importances = pd.DataFrame({
    'feature': X.columns,
    'importance': model.feature_importances_
}).sort_values('importance', ascending=False)
print(importances)

# Plot actual vs predicted
plt.figure(figsize=(12, 6))
plt.plot(y_test.values, label='Actual')
plt.plot(y_pred, label='Predicted')
plt.legend()
plt.title('Actual vs Predicted Sales')
plt.show()
