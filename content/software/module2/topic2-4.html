Topic 4: Functions and Modular Programming

Overview: Functions are fundamental to writing clean, reusable, and maintainable code. They allow us to break large problems into smaller, manageable chunks. Modular programming takes this concept further by organizing code into separate files or modules for better readability and collaboration.

1. What is a Function?

  • A block of reusable code that performs a specific task.
  • Helps reduce repetition and improves code clarity.

# A simple function
def greet():
    print("Hello, welcome to NAS Software Engineering!")

greet()

2. Parameters and Arguments

Functions can accept inputs, known as parameters, and you provide arguments when calling them.


def greet_user(name):
    print(f"Hello, {name}!")

greet_user("Ayo")

3. Return Values

Functions can return values using the return keyword, allowing further use of the result elsewhere in your code.


def add(a, b):
    return a + b

result = add(3, 5)
print(result)  # Outputs 8

4. Scope of Variables

  • Local variables exist only inside the function.
  • Global variables can be accessed throughout the program (though often discouraged).

x = 10  # global

def show_number():
    x = 5  # local
    print(x)

show_number()  # Outputs 5
print(x)       # Outputs 10

5. Lambda (Anonymous) Functions

Short functions that are defined without a name, useful for quick, one-line operations.


square = lambda x: x * x
print(square(4))  # Outputs 16

6. Modular Programming

  • Breaking programs into separate files called modules.
  • Promotes reusability, organization, and team collaboration.

# In file: math_utils.py
def multiply(a, b):
    return a * b

# In another file
from math_utils import multiply

print(multiply(3, 4))  # Outputs 12

7. Best Practices

  • Write small, single-purpose functions.
  • Use descriptive names and comments.
  • Follow consistent indentation and structure.
  • Test your functions individually.
