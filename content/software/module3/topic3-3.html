Inheritance and Polymorphism in Object-Oriented Programming

Understanding Inheritance

Inheritance is a fundamental OOP concept where a child class derives properties and behaviors from a parent class. It promotes reusability and a hierarchical relationship between classes.

Example: A Vehicle class can be a parent to subclasses like Car and Bike.

Implementing Inheritance in Python

                
# Base class (Parent)
class Vehicle:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model

    def display_info(self):
        return f"{self.brand} {self.model}"

# Derived class (Child)
class Car(Vehicle):
    def __init__(self, brand, model, doors):
        super().__init__(brand, model)  # Inherit from Vehicle
        self.doors = doors

    def car_details(self):
        return f"{self.display_info()} with {self.doors} doors"

# Creating an object of Car
car1 = Car("Toyota", "Corolla", 4)
print(car1.car_details())  # Output: Toyota Corolla with 4 doors

Types of Inheritance

  • Single Inheritance: One child class inherits from a single parent class.
  • Multiple Inheritance: A child class inherits from multiple parent classes.
  • Multilevel Inheritance: A child class inherits from another derived class.
  • Hierarchical Inheritance: Multiple child classes inherit from the same parent.

Understanding Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common superclass. This enables flexibility and dynamic method execution.

Method Overriding in Python

Method Overriding allows a child class to redefine a method from the parent class. 

                
class Animal:
    def speak(self):
        return "Some generic sound"

class Dog(Animal):
    def speak(self):  # Overriding the parent method
        return "Woof!"

class Cat(Animal):
    def speak(self):  # Overriding the parent method
        return "Meow!"

dog = Dog()
cat = Cat()
print(dog.speak())  # Output: Woof!
print(cat.speak())  # Output: Meow!

Operator Overloading

Operator Overloading allows built-in operators to have custom behaviors for user-defined objects. 

                
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):  # Overloading the '+' operator
        return Point(self.x + other.x, self.y + other.y)

p1 = Point(2, 3)
p2 = Point(4, 5)
result = p1 + p2  # Uses the __add__ method
print(result.x, result.y)  # Output: 6 8

Abstract Classes and Methods

Abstract classes provide a blueprint for other classes. Methods defined in abstract classes must be implemented in derived classes. 

                
from abc import ABC, abstractmethod

class Shape(ABC):  # Abstract Class
    @abstractmethod
    def area(self):
        pass  # Method must be implemented in derived classes

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

circle = Circle(5)
print(circle.area())  # Output: 78.5

Real-World Applications of Inheritance and Polymorphism

  • Web Development: Reusable templates for different web pages.
  • Game Development: Base character classes with unique abilities.
  • Finance: Banking applications with different account types.

Conclusion

Inheritance and Polymorphism are key OOP principles that enhance code reusability and flexibility. By using inheritance, we avoid redundancy, and with polymorphism, we achieve dynamic method execution.
