Version Control with Git and GitHub

Overview

Version control is a system that records changes to a file or set of files over time so you can recall specific versions later. Git is a popular distributed version control system, and GitHub is a cloud-based platform for hosting and managing Git repositories.

What is Git?

Git is a powerful tool for tracking changes in source code during software development. Key features of Git include:

  • Distributed Version Control: Every developer has a complete copy of the repository.
  • Branching and Merging: Create isolated branches for features or fixes and merge them later.
  • Commit History: Keep a detailed history of changes.
// Example: Common Git Commands
git init        // Initialize a new Git repository
git status      // Check the status of your repository
git add .       // Stage changes for commit
git commit -m "Initial commit" // Commit changes with a message
git log         // View commit history

What is GitHub?

GitHub is a platform for hosting and collaborating on Git repositories. It allows teams to share, review, and manage code efficiently. Key features include:

  • Repositories: Centralized storage for project code.
  • Collaboration: Teams can work together with pull requests and code reviews.
  • Integration: Tools like CI/CD, project management, and documentation.
// Example: Connect a Local Repository to GitHub
git remote add origin https://github.com/username/repository.git
git push -u origin main

Basic Git Workflow

Follow these steps to manage your code using Git:

  1. Initialize a Git repository using git init.
  2. Make changes to your code and use git add to stage them.
  3. Commit the changes with a descriptive message using git commit.
  4. Push your changes to a remote repository like GitHub using git push.

Branches and Pull Requests

Branches allow you to work on different features or bug fixes without affecting the main codebase. Once a branch is ready, create a pull request to merge it into the main branch.

// Example: Branch Workflow
git branch feature-login      // Create a new branch
git checkout feature-login    // Switch to the new branch
git push origin feature-login // Push the branch to GitHub

Collaboration Tips

  • Write Clear Commit Messages: Summarize changes in a single sentence.
  • Use Pull Requests: Review and discuss changes before merging.
  • Resolve Conflicts: Work through merge conflicts carefully.

Next Steps

To advance your skills in version control:

  • Learn about advanced Git commands (e.g., rebase, stash).
  • Explore GitHub Actions for automation.
  • Practice with open-source projects on GitHub.
