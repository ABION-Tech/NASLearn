6.2: Introduction to Node.js & npm

Introduction: JavaScript Everywhere

For years, JavaScript was confined to the browser. It was a language for making web pages interactive. Node.js shattered that limitation. It took JavaScript and placed it on the server, enabling developers to build fast, scalable network applications using a language they already knew. This module introduces you to the Node.js runtime, its powerful package manager, npm, and the core tools you need to start building.

1. What is Node.js? More Than Just JavaScript

Node.js is not a programming language nor a framework. It is a JavaScript runtime environment built on top of Google Chrome's powerful V8 JavaScript engine.

The V8 Engine

This is what makes Node.js fast. V8 is the open-source JavaScript engine that compiles JavaScript directly to native machine code inside Chrome. Node.js takes this engine and provides additional capabilities for working with the operating system, like reading files and handling networks, which a browser doesn't allow.

Running JavaScript Outside the Browser

Node.js provides the necessary APIs (e.g., fs, http, path) that allow JavaScript code to interact with the server's operating system. This means you can write a JavaScript program that starts a web server, accesses a database, or reads a configuration fileâ€”things that were previously only possible with languages like Python, Java, or PHP.

The Secret Sauce: Event-Driven, Non-Blocking I/O

This is the most important concept to grasp about Node.js. It's what makes it efficient and well-suited for I/O-heavy applications (like web servers, APIs, and chat apps).

I/O (Input/Output): Any operation that involves an external system: reading a file from a disk, waiting for a database query to return, making an HTTP request to another API. These operations are incredibly slow compared to in-memory calculations (like adding two numbers).

Blocking I/O (The Old Way): In traditional server models, when a request requires an I/O operation (e.g., reading a file), the entire process stops ("blocks") and waits idly for that operation to finish. This is inefficient, as the server could be handling other requests while it waits.

Non-Blocking I/O (The Node.js Way): Node.js uses an event loop to handle operations asynchronously.

  1. When an I/O operation is requested, Node.js doesn't wait. It registers a callback function and immediately continues to execute other code.
  2. The event loop constantly checks if any operations have completed.
  3. Once the slow I/O operation is finished, the event loop places its callback function in a queue to be executed as soon as the main thread is free.

The Analogy: Imagine a single restaurant waiter (the event loop) serving many tables (requests).

  • Blocking: The waiter takes an order from Table 1, goes to the kitchen, and stands there waiting for the food to be cooked before serving Table 2. This is terribly inefficient.
  • Non-Blocking: The waiter takes an order from Table 1 and gives it to the kitchen. They immediately go to Table 2 to take their order. When the kitchen signals that Table 1's food is ready, the waiter picks it up and delivers it. One waiter efficiently handles many tables.

This model allows a single-threaded Node.js application to handle thousands of concurrent connections efficiently, making it lightweight and performant.

2. Setting Up and Running Node.js

Installation
  1. Go to the official Node.js website.
  2. Download and run the installer for your operating system (Windows, macOS, Linux). The LTS (Long Term Support) version is recommended for most users as it is more stable.
  3. The installer includes both Node.js and npm (Node Package Manager).
Verifying the Installation

Open your terminal or command prompt and run:

node --version
npm --version

You should see version numbers printed out (e.g., v18.17.0 and 9.6.7). This confirms a successful installation.

Running a JavaScript File

Create a file named app.js with the following content:

// app.js
console.log('Hello from Node.js!');
const message = "The answer is";
const answer = 42;
console.log(`${message} ${answer}`);

In your terminal, navigate to the directory containing app.js and run:

node app.js

You will see the output: Hello from Node.js! followed by The answer is 42.

The Node.js REPL (Read-Eval-Print Loop)

A REPL is an interactive programming environment. You can launch it by simply typing node in your terminal (without a filename).

$ node
> 2 + 2
4
> const name = 'Your Name';
undefined
> console.log(`Hello, ${name}!`);
Hello, Your Name!
undefined
> .exit

This is useful for quickly testing small snippets of JavaScript code without creating a file.

3. npm (Node Package Manager): The World's Largest Software Registry

npm is two things: 1) an online repository of hundreds of thousands of free packages (libraries) of code, and 2) a command-line utility installed with Node.js to interact with that repository.

The package.json File

This is the manifest file for your Node.js project. It holds metadata about your project and, most importantly, a list of all the external packages (dependencies) your project needs to run. You can create one manually or let npm create it for you.

# Navigate to your project's root directory and run:
npm init -y

The -y flag tells npm to use default values for all prompts, generating a standard package.json.

Installing Packages

Local Install (The most common): Packages are installed in a node_modules folder inside your project directory. They are only available for that specific project.

# Syntax: npm install <package-name>
npm install express

This installs the popular Express.js web framework and adds it to the "dependencies" section of your package.json. Other developers can now run npm install (with no arguments) to download all the dependencies listed in your package.json.

Global Install (-g): Packages are installed system-wide, making their command-line tools available anywhere. These are typically utilities, not project libraries.

npm install -g nodemon

nodemon is a tool that automatically restarts your Node.js application when it detects file changes, which is invaluable for development.

Key npm Commands
Command Purpose
npm init Creates a package.json file interactively.
npm init -y Creates a package.json with default values.
npm install <pkg> Installs a package locally and adds it to dependencies.
npm install --save-dev <pkg> Installs a package and adds it to devDependencies (for tools like linters, test frameworks).
npm install -g <pkg> Installs a package globally.
npm uninstall <pkg> Removes a locally installed package.
npm update Updates all packages to their latest version.
npm list Lists all installed packages in the current project.
npm list -g --depth=0 Lists all globally installed top-level packages.

4. Exploring Built-In Modules: Node.js's Standard Library

Node.js comes with a rich set of built-in modules that provide core functionality. You don't need to install them; you just need to require them in your code.

The fs (File System) Module

Allows you to work with the file system. All methods come in both synchronous (blocking) and asynchronous (non-blocking, callback-based) forms. Prefer the asynchronous versions in a real server.

const fs = require('fs'); // require() is the traditional way to import modules

// Asynchronously read a file (Non-Blocking)
fs.readFile('myfile.txt', 'utf8', (err, data) => {
  if (err) {
    console.error("Error reading file:", err);
    return;
  }
  console.log("File content:", data);
});
console.log("This will run BEFORE the file is finished reading!");

// Synchronously read a file (Blocking - avoid in production servers!)
try {
  const data = fs.readFileSync('myfile.txt', 'utf8');
  console.log("File content:", data);
} catch (err) {
  console.error("Error reading file:", err);
}
console.log("This will run AFTER the file is finished reading.");
The http Module

The core module for creating HTTP servers and making HTTP requests. It's low-level but powerful.

const http = require('http');

// Create a basic server
const server = http.createServer((req, res) => {
  // This function runs every time a request is received
  console.log(`Request received for: ${req.url}`);
  // Set the response header
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  // Send the response body
  res.end('Hello from your first Node.js server!\n');
});

// Tell the server to listen on port 3000
server.listen(3000, () => {
  console.log('Server is running on http://localhost:3000');
});

Run this script with node, then open your browser and go to http://localhost:3000. You've just built a web server!

The path Module

Provides utilities for working with file and directory paths. It's essential for handling cross-platform path differences (Windows uses \, macOS/Linux use /).

const path = require('path');

const myFilePath = '/users/joe/docs/letter.txt';

// Get the base filename
path.basename(myFilePath); // 'letter.txt'

// Get the directory name
path.dirname(myFilePath); // '/users/joe/docs'

// Get the file extension
path.extname(myFilePath); // '.txt'

// Join path segments, handling slashes correctly
path.join('/users', 'joe', 'docs', 'report.pdf'); // '/users/joe/docs/report.pdf'

// Create an absolute path from relative segments
path.resolve('docs', 'text.txt'); // e.g., '/current/working/directory/docs/text.txt'

Key Takeaway

Node.js, combined with npm, provides a powerful and efficient platform for building server-side applications. Its event-driven, non-blocking architecture is its superpower, allowing you to handle many users with minimal resources. The vast ecosystem of npm packages means you rarely have to build anything from scratch.
