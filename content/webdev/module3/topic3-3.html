Page 3: Using Media Queries for Responsive Design

Introduction to Media Queries

Media queries are a fundamental part of responsive web design, allowing developers to apply different styles based on screen size, resolution, or device type. This helps create websites that adjust seamlessly across desktops, tablets, and mobile devices.

Key Features of Media Queries:

  • Enable device-specific styling.
  • Allow changes in layout and visibility.
  • Support responsive typography and images.

Basic Syntax of Media Queries

A media query consists of a CSS rule that applies only when a specific condition is met, such as screen width.

Example: Basic Media Query

            
/* Default styling for large screens */
body {
    font-size: 18px;
}

/* Apply styles for screens smaller than 768px */
@media screen and (max-width: 768px) {
    body {
        font-size: 16px;
    }
}

Creating a Fully Responsive Layout

By combining **Flexbox/Grid** with media queries, we can build a fully responsive design that adjusts based on screen size.

Example: Responsive Two-Column Layout

            
.container {
    display: flex;
    gap: 20px;
}

/* Make it a single column on smaller screens */
@media screen and (max-width: 600px) {
    .container {
        flex-direction: column;
    }
}

Media Queries for Different Devices

Device Type Screen Width Example Media Query
Mobile Up to 600px @media screen and (max-width: 600px) { ... }
Tablet 601px - 1024px @media screen and (min-width: 601px) and (max-width: 1024px) { ... }
Desktop 1025px and above @media screen and (min-width: 1025px) { ... }

Summary

Media queries allow developers to create flexible designs that work across multiple devices. By adjusting typography, layout, and visibility based on screen width, we can enhance the user experience.
